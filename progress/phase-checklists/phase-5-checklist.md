# Phase 5 Checklist: Context Resolution Engine

## Checklist

- [x] **Context Merging Module**: Implemented `context_resolver.py` inside `apps/api/app/nlq/` to identify rule-based follow-up patterns ("break down by", "compare", "show as").
- [x] **Determinism**: Verified through tests that context merging relies solely on regex pattern matching and `DatasetColumn` schema inspection. No LLM calls are invoked during deterministic merging.
- [x] **Fallback Handling**: Unrecognized or unsupported follow-ups elegantly return `treat_as_fresh`, triggering a normal LLM intent resolution.
- [x] **State Persistence**: Verified `chart_spec["meta"]["intent"]` persistence allows extracting the fully typed `AnalyticsIntent` from prior interaction state without requiring dedicated backend state machines.
- [x] **Chained Resolution**: Confirmed that context variables gracefully propagate through 4-turn dialogues.
- [x] **Dynamic Followup UI Helpers**: Auto-generation of 2-3 contextual heuristics (`suggested_followups`) directly from unused dimensions without calling an LLM.
- [x] **Chart Type Hinting**: Added support for respecting `chart_hint` generated by the resolver to seamlessly convert query results into the expected visual formats.

## Testing Log
